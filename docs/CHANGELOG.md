## 2026-07-21

**Инфраструктура**
- Задеплоено на VPS Latvia (`31.76.0.133`) как `certified-loverboy.service`, порт 3101, за nginx на `https://vacanator.xyz:8443/`. Порт 443 занят VPN-стеком сервера (Amnezia Xray) — туда не лезем, HTTPS у нас на 8443.
- `scripts/deploy.sh` — деплой одной командой, сам решает какие шаги нужны (см. README).
- Домен `vacanator.xyz` — ранее в памяти числился как занятый другим проектом, по факту сервер был чист (nginx даже не стоял), конфликта нет.

**Данные / модель**
- Свиданка (`DateIdea`) больше не хранит адрес/метро/координаты напрямую — вынесено в отдельную таблицу `Location` (один-ко-многим), у одной идеи может быть несколько мест, у каждого свои address/metro/lat/lng/url.
- Убраны поля `inPartnerDeck`/`showPriceToPartner` — раньше были чекбоксы ручного управления видимостью, оказались лишними: теперь ВСЕ свиданки всегда видны партнёрше, а цена ей — всегда скрыта (без исключений).
- Добавлен `DateIdeaType` (`DATE`/`FOOD`) и `Match.isFavorite` (Codex).
- `swipeDescription` — отдельное короткое поле специально для карточки свайпа (Codex), не путать с обычным `description`.

**Auth**
- Полностью без пароля — Telegram Mini App `initData`, роль по telegram user id. Debug-режим вне Telegram: `?debug_init=<initData>`.

**UI**
- Дизайн-система на CSS-переменных (`--app-*`), светлая/тёмная тема, иконки lucide-react вместо эмодзи.
- Карта: пробовали OpenFreeMap/MapLibre (векторные стили), Codex откатил обратно на обычный OpenStreetMap TileLayer — см. `src/components/mapInternals.tsx` для актуального состояния.
- Мульти-select фильтры (теги/метро) — управляемые родителем, чтобы не открывались одновременно.
- Нижний таббар — `position: fixed`, раньше терял это при редизайне и уезжал за экран на длинных страницах.
- Импорт из markdown теперь поддерживает несколько мест на файл (маркеры `Место:`/`Локация:`/`Точка:`, см. `src/lib/parseDateMarkdown.ts`).

**Известные грабли**
- `prisma migrate dev` — shadow-db валидация может падать на этой истории миграций (одна из ранних миграций была сгенерирована раньше по времени, чем должна была логически идти). Рабочий обход: писать миграцию руками и накатывать через `prisma migrate deploy` (не `dev`), которая не делает full-replay с нуля.
- Если `npm run build` падает с "Another next build process is already running" — это скорее всего осиротевший `.next/turbopack` lock-файл от прерванного билда, не реальный конфликт процессов. `rm .next/turbopack` и пересобрать.
- SQLite поддерживает `ALTER TABLE ... DROP COLUMN` напрямую (современная версия в better-sqlite3) — не обязательно городить полный RedefineTables (create new table + copy + rename), которым иногда грешит автогенератор Prisma.
